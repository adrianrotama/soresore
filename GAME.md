# Soresore ‚Äî Game architecture & handoff

> **For AI agents:** Read this before changing multiplayer, movement, or camera code. Visual/MVP direction: `AGENTS.md`. Next.js quirks: `node_modules/next/dist/docs/`.

### Doc maintenance (agents)

- **Do not edit this file during normal work.** Update `GAME.md` only when the user explicitly asks (typically end-of-day handoff). Code, `lib/stationLayout.js`, and `lib/tileModels.js` are the live source for tuned numbers.
- **Do not ‚Äúfix‚Äù `SYNC_MS` in `Game.js`.** It is intentionally `50000` (50s) to limit Supabase writes while prototyping. Raising sync rate is a deliberate product decision, not a bug.

### Project rules (read before extending)

- **Cozy direction wins (`AGENTS.md`).** ACNH-inspired pedestal silhouette, sunset palette, stable camera. If a refactor makes the world feel less cozy, push back ‚Äî challenge the request.
- **Tile pedestal is the world unit anchor.** `TILE_SIZE = 2` m XZ; `TILE_LEVEL_HEIGHT = 1.0` m Y; mesh footprint **2.082 m** (slight bleed for seamless joints). Scale OTHER assets to fit the grid, not the other way around (e.g. tune `TRAIN_SCALE`, not `TILE_SIZE`).
- **One pedestal GLB; palette via vertex-color repaint** (`TileModel`). Don't duplicate GLBs per color. New tile **type** = new entry in `TILE_PALETTES`. New tile **shape** (sunken water, slope) = new GLB only.
- **No PBR textures on tiles.** `MeshToonMaterial` + vertex colors only. Texture atlases break the unified palette system.
- **`surfaceYAt(world, gx, gz)` is the only source of ground Y.** Decorations, player snap (Phase 4), and walkability rules (Phase 5) read from it. Never hard-code Y for tile-based content.
- **Decoration vs Landmark.** Grid-anchored small props ‚Üí `component/Decoration.js` (`{ kind, gx, gz }`). World-anchored large structures ‚Üí `component/Landmark.js` (`{ kind, position?, ... }`). Don't put a tree in landmarks or a depot in decorations.
- **All world content lives in one object** (`TEST_WORLD` in `lib/testWorld.js`). `<World data={‚Ä¶} />` is the single scene root for world content. No standalone `<StationRailProps>` etc. ‚Äî wrap as a landmark kind.
- **Blender conventions** for new tile/decoration GLBs: 2.082 √ó 2.082 √ó 1.0 m for pedestals; bottom-center origin; vertex colors (top vs side) ‚Äî `TileModel` Y-threshold classifier will detect. Export to `public/images/tiles/`. Props at feet-origin so `cellSurfaceWorld` drops them on tiles.
- **Depot anchor** `STATION_START.y = TILE_LEVEL_HEIGHT`. Helpers (`stationHutWorldPosition`, `getPlayerSpawn`, `platformLayout`) read `start[1]`. Don't hardcode `y = 0` in new station helpers.

Browser multiplayer 3D prototype. One tab = one player. Positions sync via **Supabase**; remotes render as cubes. Target: cozy low-poly world (`AGENTS.md`), not a debug scene.
‚ÄúTiny cozy world‚Äù. Phase 1 Features: movement, chat, diary, friend.

## Supabase `players` table

| Column | Notes |
|--------|--------|
| `id` | uuid PK, client-generated per tab |
| `x`, `y`, `z` | World position; local spawn `y: 0.5` |
| `last_seen` | Heartbeat timestamp |

- Enable **Realtime** on `players`.
- RLS: anon `SELECT` + upsert. **No DELETE** ‚Äî offline = stale `last_seen` + client prune.
- Channel `players-room`: client name for `postgres_changes` on `players`.

## Timing (`Game.js`)

| Constant | Value | Role |
|----------|-------|------|
| `SYNC_MS` | **50000** | Local upsert interval ‚Äî **intentional** (reduce DB load); agents must not change to 5000 without user approval |
| `STALE_MS` | 15000 | Drop if `last_seen` older than this (`PRUNE_MS` sweep) |
| `PRUNE_MS` | 2000 | Sweep stale ids from in-memory state |

Prune only clears React state, not DB rows.

**Note:** With `SYNC_MS` > `STALE_MS`, `last_seen` only advances on upsert, so remotes may prune quickly until sync timing is retuned together. Acceptable for low-traffic prototype; fix `STALE_MS` (e.g. > `SYNC_MS`) when multiplayer presence matters.

## Local player (`LocalPlayer.js`)

**State:** `positionRef` in `Game.js` (mutable, no per-frame React updates). Mesh via `meshRef` in `useFrame`.

**Movement:** camera-relative WASD ‚Üí world direction ‚Üí velocity steers toward target with separate **accel** / **decel** (`1 - Math.exp(-rate * delta)`). Position: `pos += vel * delta`.

**Camera basis:** `forward` = horizontal camera‚Üíplayer; `right` = `forward √ó up` ‚Üí `{ x: -forward.z, z: forward.x }` (wrong cross order reverses strafe).

**Rotation:** yaw toward velocity; on reversal (`dot(vel, input) < 0.25`) face **input**; `MAX_TURN_SPEED` caps snap on 180¬∞ turns.

**Input safety:** clear keys on `blur` and `visibilitychange` (hidden).

**Tuning:** `MAX_SPEED`, `ACCELERATION`, `DECELERATION`, `ROTATION_SMOOTH`, `MAX_TURN_SPEED` at top of file.

**Visual ‚Äúphysical lies‚Äù (local only ‚Äî does not affect `positionRef` / Supabase):**

- `movePhaseRef`: continuous phase; advances at `MOVE_BOB_HZ` (default `2.0`).
- `moving01`: smoothed movement intensity in **0..1** (`moveIntensityRef` lerps toward `speed / MAX_SPEED`). Used for bob/squash strength and footstep gating/volume. Name is shader-style shorthand; consider renaming to `walkIntensity` if refactoring.
- Idle bob: `sin(time * IDLE_BOB_HZ)`; move bob: `sin(movePhase) * MOVE_BOB_AMP * moving01`.
- Squash: `(1 - cos(movePhase))` scaled by `moving01`.
- **Start/stop jitter fix:** constant `MOVE_BOB_HZ` + phase accumulator (do not multiply `time` by a per-frame changing frequency).

**Footstep ticks:** `consumeStepTicks(movePhase, ‚Ä¶)` in `lib/gameAudio.js` ‚Äî one sound per **full bob cycle** (`STEP_PHASE_RADIANS = 2œÄ`). Do not use `œÄ` unless you intentionally want two ticks per bob.

## Audio

**Assets (`public/audio/`):**

| Path | Use |
|------|-----|
| `ambiances/abeto-base.ogg` | Looping ambience (fades in after unlock) |
| `player/footstep-wood.wav` | Reserved for surface zones (not wired yet) |
| `player/footstep-grass.wav` | Short one-shot; default wood surface (preferred) |
| `player/abeto-footsteps.ogg` | Long clip ‚Äî **do not** use for per-step ticks; trim or ignore |

**Flow:**

1. `GameAudio` ‚Äî first `keydown` or `pointerdown` calls `unlockGameAudio()` (browser autoplay policy).
2. `lib/gameAudio.js` ‚Äî Web Audio API: decode buffers, loop ambience with gain ramp, `playFootstep()` one-shots.
3. `LocalPlayer` ‚Äî each frame `consumeStepTicks(movePhase, lastStepIndexRef, moving01)` after advancing phase.

**Tuning (`lib/gameAudio.js`):**

| Constant | Role |
|----------|------|
| `FOOTSTEP_VOLUME` | One-shot level |
| `AMBIENCE_VOLUME` | Loop bed level |
| `MIN_MOVING_FOR_STEP` | `moving01` threshold: no steps below this; resets step index while idle/slow to avoid catch-up bursts on resume |
| `STEP_PHASE_RADIANS` | `2œÄ` = one footstep per visual bob cycle |

**Note:** `ensureLoaded()` currently decodes `AUDIO.footstepGrass` into the active footstep buffer. Switch to `footstepWood` when wood is the default again. Surface-based switching = future work (zones + `footstepGrass`).

**Tab hidden:** `setAudioMuted(true)` suspends context and fades ambience out.

## Tile system & world (`component/World.js`)

The world model. Replaces ad-hoc placement: every object asks the world "where's the ground at my cell?" via `surfaceYAt(world, gx, gz)`. Single mount: `<World data={TEST_WORLD} positionRef={‚Ä¶} />` in `Game.js`.

### Architecture

```
world data    { map, origin, decorations, landmarks }
                          ‚îÇ
                          ‚ñº
<World>       TileMap + Decoration[] + Landmark[]
                          ‚îÇ
                          ‚ñº
surfaceYAt    single source of truth for ground Y (decorations + future player snap)
```

### Tile mesh contract

- **Footprint:** 2.082 √ó 2.082 m on XZ (~4% bleed over `TILE_SIZE = 2` ‚Äî hides corner seams from rounded Kenney tops).
- **Height:** 1.0 m (`TILE_LEVEL_HEIGHT`).
- **Pivot:** bottom-center, origin on ground (mesh top at `+1`).
- **Vertex colors only:** no PBR. `TileModel` discards the original `colormap` texture and bakes fresh vertex colors per palette at load.

### Palette per tile type (one GLB, many types)

Source: `public/images/tiles/kenney_platformer-kit/block-grass-large.glb` (Kenney pedestal, scale 1:1).

- `TileModel` clones geometry per cell, classifies vertices by Y position (`TILE_TOP_Y_RATIO = 0.8`), paints top/side colors.
- Material: `MeshToonMaterial` with `vertexColors: true` ‚Äî stepped cel-shading; sides don't crush to black under warm sunset light.
- Types in `lib/tileModels.js` `TILE_PALETTES`: `grass`, `path`, `sand`, `dirt`, `stone`. Side stays unified tan for grass/path/sand so the silhouette reads as one continuous bank.

### Decorations vs Landmarks

| Concept | Anchor | Example | Registered in |
|---------|--------|---------|---------------|
| **Decoration** | grid (`gx, gz`) ‚Äî auto-follows elevation via `surfaceYAt` | tree, sign, marker | `component/Decoration.js` `PROP_COMPONENTS` |
| **Landmark** | world (`position`) ‚Äî hand-placed, complex | depot, konbini, river | `component/Landmark.js` `LANDMARK_COMPONENTS` |

Both registered together in `world` data (`testWorld.js`). `World` renders both. Add a new kind: register a component, declare in world data ‚Äî no other changes.

### Depot integration

`STATION_START.y = TILE_LEVEL_HEIGHT` (1.0). Helpers (`stationHutWorldPosition`, `getPlayerSpawn`, `platformLayout`) now respect `start[1]`. Train + rails + hut + spawn all sit on the tile pedestal at level 0. Depot is wrapped as `{ kind: "depot" }` landmark.

### `surfaceYAt` contract

```js
surfaceYAt(world, gx, gz) ‚Üí (cell.level + 1) * TILE_LEVEL_HEIGHT
```

- Level 0 ‚Üí Y = 1.0 (top of one pedestal)
- Level N ‚Üí Y = (N+1) * 1.0 (Phase 2+ ‚Äî not rendered yet)
- Out of map ‚Üí Y = 0 (ambient ground)

Use `cellSurfaceWorld(world, gx, gz)` to get the full `[x, y, z]` for placing a prop.

### Not done (phases ‚Äî see Handoff for priority)

- **Phase 2 ‚Äî Elevation rendering:** position each tile at `cell.level * TILE_LEVEL_HEIGHT`.
- **Phase 3 ‚Äî Slopes:** ramp tile type connecting two levels (4 rotations).
- **Phase 4 ‚Äî Player ground-snap:** `LocalPlayer` reads `surfaceYAt` for its cell.
- **Phase 5 ‚Äî Walkability rules:** `canMoveTo(world, from, to)` blocks cliff jumps; allows slope-adjacent.
